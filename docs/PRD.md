# PRD - خلّيها ترند

## 1. ملخص المنتج

«خلّيها ترند» منصة عراقية تابعة إلى SA Studio تربط العلامات التجارية والتجار بصناع المحتوى. تنشئ العلامة التجارية حملة ممولة، وينضم إليها صانع المحتوى المؤهل، ثم ينشر المحتوى على حساباته الخارجية في TikTok أو Instagram أو Facebook أو YouTube، ويرسل رابط المنشور إلى المنصة لمراجعته واحتساب المشاهدات المؤهلة والأرباح.

المنصة ليست شبكة اجتماعية ولا تستضيف منشورات عامة داخلية. دورها هو إدارة الحملات، الملفات، الشروط، الروابط، المراجعات، الإحصائيات، الأرباح، المحافظ، السحب، النزاعات، ومكافحة الاحتيال.

## 2. الأهداف

- إطلاق MVP قابل للتطوير كـModular Monolith.
- دعم اللغة العربية وواجهة RTL أولاً، مع تجهيز البنية للإنكليزية LTR.
- توفير REST API موثق يمكن استخدامه لاحقاً من تطبيق Mobile.
- منع الحسابات المالية الوهمية عبر Ledger حقيقي وقابل للتدقيق.
- بناء Provider Interfaces للتكاملات الاجتماعية والمالية دون ادعاء وجود تكاملات حقيقية قبل توفر الموافقات والاعتمادات.

## 3. خارج النطاق في MVP

- نشر الفيديوهات داخل المنصة.
- شبكة اجتماعية داخلية أو feed محتوى.
- تكامل دفع حقيقي مع ZainCash أو FastPay دون وثائق رسمية وCredentials وSandbox.
- جلب metrics من منصات التواصل بطرق scraping أو APIs غير رسمية.
- نظام AI متقدم لكشف الاحتيال. MVP يستخدم قواعد واضحة ومراجعة بشرية.

## 4. المستخدمون

### الزائر

- مشاهدة الصفحة الرئيسية.
- تصفح حملات عامة مختارة.
- قراءة صفحات كيف تعمل المنصة، الشروط، الخصوصية، وسياسات الدفع.
- التسجيل كصانع محتوى أو علامة تجارية.

### صانع المحتوى

- إنشاء ملف شخصي.
- ربط حسابات اجتماعية.
- تصفح الحملات والانضمام عند الأهلية.
- تحميل ملفات الحملة.
- إرسال رابط فيديو منشور خارجياً.
- متابعة حالة المراجعة والمشاهدات المؤهلة والأرباح.
- إدارة المحفظة وطلبات السحب.
- فتح اعتراضات ونزاعات.

### العلامة التجارية

- إنشاء ملف علامة تجارية.
- طلب توثيق.
- إنشاء حملات وشروطها وملفاتها وأسعار CPM.
- تمويل الميزانية يدوياً في MVP.
- مراجعة المشاركات والفيديوهات.
- متابعة المصروفات والتحليلات والنزاعات.

### المشرف

- مراجعة المستخدمين والعلامات والحملات والإرسالات.
- إدخال metrics يدوياً في MVP مع سبب إلزامي.
- اعتماد الإيداعات والسحوبات.
- مراجعة الاحتيال والنزاعات.
- إدارة الإعدادات والسياسات وFeature Flags.

### Super Admin

- صلاحيات النظام العليا.
- لا ينشأ عبر التسجيل العام.
- يستخدم للإعداد الأولي وإدارة المشرفين والصلاحيات الحساسة.

### رحلة صانع المحتوى

4. الانضمام لحملة بعد تحقق الأهلية.
5. تحميل أصول الحملة وقراءة الشروط.
6. نشر الفيديو خارج المنصة.
7. إرسال الرابط.
8. مراجعة الرابط والفيديو.
9. إدخال أو جلب metrics.
10. احتساب الأرباح وتحريرها بعد فترة تحقق.
11. طلب سحب.

### رحلة العلامة التجارية

1. التسجيل واختيار دور علامة تجارية.
2. إنشاء ملف العلامة.
3. إنشاء حملة عبر wizard.
4. إرسال الحملة لمراجعة الإدارة.
5. إنشاء طلب إيداع وتمويل يدوي.
6. حجز ميزانية للحملة.
7. إطلاق الحملة.
8. مراجعة المشاركات والإرسالات.
9. متابعة المصروفات والتحليلات.

### رحلة الإدارة

1. مراجعة الحملات قبل النشر.
2. اعتماد الإيداعات.
3. مراقبة الإرسالات والاحتيال.
4. إدخال metrics يدوياً عند الحاجة.
5. مراجعة السحوبات والنزاعات.
6. تدقيق Audit Log والعمليات المالية.

## 6. المتطلبات الوظيفية

- مصادقة وصلاحيات RBAC على السيرفر.
- ملفات شخصية لصناع المحتوى والعلامات التجارية.
- ربط حسابات اجتماعية عبر Manual Provider في MVP.
- نظام حملات بحالات واضحة.
- Campaign Terms Snapshot عند الانضمام.
- رفع ملفات آمن عبر Signed URLs.
- إرسال روابط المنشورات مع URL normalization.
- منع الرابط أو Platform Post ID المكرر.
- Metrics Snapshot غير قابل للتعديل.
- محرك أرباح يعتمد على qualified views والميزانية والحدود.
- Double-entry Ledger أو تصميم مكافئ متوازن.
- محفظة وطلبات سحب يدوية.
- نظام نزاعات واعتراضات.
- Fraud Score وTrust Score مبدئيان.
- إشعارات داخل المنصة.
- API versioned تحت `/api/v1`.
- PWA للصفحات العامة وواجهة Mobile First.

## 7. المتطلبات غير الوظيفية

- TypeScript strict.
- اختبارات وحدة وتكامل وE2E للرحلات الحرجة.
- أمان ضد IDOR وBroken Access Control وXSS وCSRF ورفع الملفات الضارة.
- Audit Log لكل العمليات المالية والحساسة.
- Idempotency للعمليات المالية.
- Pagination وFiltering وSorting في API.
- عدم تخزين أسرار في المستودع.
- عدم استخدام بيانات حقيقية في seed أو الاختبارات.

## 9. معايير القبول العامة

- لا يستطيع صانع المحتوى إنشاء رصيد لنفسه.
- لا يستطيع التاجر تعديل المشاهدات النهائية.
- لا يمكن الوصول إلى بيانات مستخدم آخر دون صلاحية.
- لا يمكن إرسال الرابط نفسه مرتين.
- لا يمكن احتساب snapshot مرتين.
- لا يمكن تجاوز حد أرباح الفيديو أو ميزانية الحملة.
- لا يمكن سحب رصيد قيد التحقق.
- لا يمكن دفع طلب سحب مرتين.
- كل حركة مالية لديها Audit Trail.
- شروط الحملة بعد الانضمام لا تؤثر بأثر رجعي على المشاركين السابقين.
- التطبيق يعمل RTL بالكامل على الهاتف.
